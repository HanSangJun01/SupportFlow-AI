<must_haves>
- D-01/D-02/D-03: Tests and docs must show only shared-shape `FAQ` and `POLICY` documents with statuses `ACTIVE` and `ARCHIVED`.
- D-04/D-05: Tests must prove create, update, archive, and restore require explicit `actorUserId` and reject cross-tenant or inactive actors.
- D-06/D-07: Tests and docs must show overwrite update semantics, `createdByUserId`, `updatedByUserId`, `createdAt`, and `updatedAt`; no immutable revisions.
- D-08: Tests must prove archived metadata updates are allowed, `type` can change while archived, and archived content updates are rejected.
- D-09: Tests must prove list defaults exclude archived documents and exact `status=ARCHIVED` includes archived documents.
- D-10/D-11/D-12/D-13: Tests and docs must cover `sourceLabel`, optional `sourceUrl`, normalized tags, `effectiveFrom`, `effectiveTo`, and server-computed `contentHash`.
- D-14/D-15/D-16: OpenAPI and SDD must document `/api/v1/tenants/{tenantId}/knowledge-documents` create, list, detail, update, archive, restore, plus exact filters `type`, `status`, `tag`, and `activeAt`.
- D-17: Tests and docs must explicitly exclude AI classification, RAG, embeddings, vector search, Mongo text search, keyword contains search, frontend UI, authentication, full RBAC, destructive delete, and immutable revisions.
- Locked defaults: Mongo integration tests must prove archive and restore are idempotent 200 operations, restore clears archive metadata, and archived `type` updates are allowed while archived `content` updates fail.
</must_haves>

<threat_model>
| Threat ID | STRIDE | Component | Disposition | Mitigation Plan |
|-----------|--------|-----------|-------------|-----------------|
| T-03-02-01 | Information Disclosure | Cross-tenant document and actor IDs | mitigate | Add Mongo-backed HTTP tests proving tenant A cannot read, update, archive, or restore tenant B documents and cannot use tenant B actors. |
| T-03-02-02 | Tampering | Inactive tenant mutation behavior | mitigate | Add Mongo-backed tests proving inactive tenants can list/detail knowledge documents but reject create, update, archive, and restore with HTTP 409. |
| T-03-02-03 | Scope Creep | Retrieval/search API contract | mitigate | Add OpenAPI and SDD assertions that document only exact metadata filters and explicitly exclude search, text indexes, embeddings, RAG, AI, frontend, auth, RBAC, delete, and revisions. |
| T-03-01-01 | Tampering | Content hash integrity | mitigate | Add validation tests proving client-supplied hashes are ignored or impossible and that hash changes only when stored content changes. |
| T-03-01-03 | Tampering | Archived document mutation | mitigate | Add integration and service tests proving archived content updates fail while metadata updates, including `type`, succeed. |
</threat_model>
