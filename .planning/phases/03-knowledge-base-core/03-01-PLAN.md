<must_haves>
- D-01: `KnowledgeDocumentType` must be a validated enum with exactly `FAQ` and `POLICY`.
- D-02: FAQ and policy documents must share `title` and `content` fields; do not add FAQ-only question/answer fields or policy-only body fields.
- D-03: `KnowledgeDocumentStatus` must be limited to `ACTIVE` and `ARCHIVED`.
- D-04/D-05: Every mutation request must require explicit `actorUserId` and validate it with `OperationalUserService.validateActiveActor(tenantId, actorUserId)`.
- D-06/D-07: Updates overwrite current fields, refresh `updatedAt`, and track `createdByUserId` plus `updatedByUserId`; do not create immutable revisions.
- D-08: Archived documents may receive metadata and status updates, including restore, but content updates while `ARCHIVED` must fail.
- D-09: List responses must exclude archived documents by default and support an optional exact `status` filter.
- D-10/D-11/D-12/D-13: Documents must store required `sourceLabel`, optional `sourceUrl`, normalized optional `tags`, optional `effectiveFrom` and `effectiveTo`, and backend-computed `contentHash`.
- D-14/D-15: Routes must live under `/api/v1/tenants/{tenantId}/knowledge-documents` and expose create, list, detail, update, archive, and restore operations.
- D-16: List filtering must support exact `type`, exact `status`, exact normalized `tag`, and an `activeAt` date-window query parameter.
- D-17: Do not add text search, Mongo text indexes, keyword contains search, embeddings, vector search, AI/RAG behavior, frontend UI, authentication, full RBAC, destructive delete, or document revisions.
- Locked defaults: archive and restore endpoints are idempotent 200 operations; restoring clears `archivedAt` and `archivedByUserId`; `type` may be updated while active or archived, but archived `content` changes are rejected.
</must_haves>

| T-03-01-02 | Information Disclosure | Tenant-scoped knowledge routes | mitigate | Read/list/detail/update/archive/restore must use `tenantId` path identity and `findByTenantIdAndId`; never use repository `findById` for tenant routes. |
| T-03-01-03 | Tampering | Archived document mutation | mitigate | Reject `content` updates when status is `ARCHIVED`; allow metadata updates including `type`, `title`, `sourceLabel`, `sourceUrl`, `tags`, `effectiveFrom`, and `effectiveTo`. |
| T-03-02-03 | Scope Creep | Retrieval/search behavior | mitigate | Keep this plan to document registration and metadata APIs only; do not add text search, indexes, embeddings, AI calls, RAG retrieval, frontend, auth, RBAC, delete, or revisions. |
</threat_model>
